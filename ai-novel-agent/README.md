# ai-novel-agent

用 CodeBuddy Agent 辅助写网文的工具链，主要解决长篇创作中伏笔遗忘、情节矛盾、手动发布低效的问题。

## 怎么运作

续写时 Agent 按「读-写-校-更」循环工作：

1. **写前读**：读伏笔文档 → 读大纲状态 → 读前一章，保证衔接
2. **写章节**：2500-3000 字，有文风约束（段落长度、表述规范等）
3. **写后自检**：扫一遍违规项和节奏
4. **写后更新**：更新伏笔和大纲文档，每 3 章迭代一次

发布用 Playwright + CDP 跑脚本，自动在番茄小说后台建章节、粘正文、点发布，支持批量。

## 实际情况

不算完善的产品，更像给自己搭的工作流：

- 伏笔/大纲文档的维护大部分靠 Agent 自动完成，但偶尔还是要人工检查修正
- 发布脚本能跑，但番茄页面改版就可能出问题，需要跟着调
- 文风约束有用但不是万能的，写出来还是经常需要手动润色
- 目前写了约 197 章 / 60 万字

## 技术栈

- CodeBuddy (WorkBuddy) Agent / GLM 模型
- Playwright + CDP（浏览器自动化）
- Markdown 结构化文档
